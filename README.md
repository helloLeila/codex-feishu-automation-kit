# Codex Feishu Automation Kit

Send Markdown output from Codex automations to Feishu/Lark custom bots as interactive cards.

This kit is designed for recurring Codex jobs that generate Markdown reports, such as AI industry dailies, weekly digests, event searches, or internal monitors.

## What Is Included

```text
skills/feishu-automation-reporter/
  SKILL.md
  scripts/
    push-ai-daily-to-feishu.mjs
    push-gba-events-to-feishu.mjs
examples/
  .env.local.example
  ai-daily-example.md
  gba-events-example.md
docs/
  codex-automation-setup.md
```

## Requirements

- Node.js 18+.
- A Feishu/Lark custom bot webhook.
- A Markdown file generated by a Codex automation or another workflow.

The scripts use only Node.js built-ins and the global `fetch` available in modern Node.js.

## Configure Secrets

Copy the example env file:

```bash
cp examples/.env.local.example .env.local
```

Edit `.env.local`:

```bash
FEISHU_WEBHOOK_URL="https://open.feishu.cn/open-apis/bot/v2/hook/replace-me"
```

If the bot enables signature verification:

```bash
FEISHU_WEBHOOK_SECRET="replace-me"
```

`.env.local` is intentionally ignored by git.

For worktree or CI runs, you can also point the scripts at a specific env file:

```bash
FEISHU_ENV_FILE=/absolute/path/to/.env.local node skills/feishu-automation-reporter/scripts/push-ai-daily-to-feishu.mjs examples/ai-daily-example.md
```

## Push An AI Daily

```bash
node skills/feishu-automation-reporter/scripts/push-ai-daily-to-feishu.mjs examples/ai-daily-example.md
```

Dry-run without sending:

```bash
FEISHU_DRY_RUN=1 node skills/feishu-automation-reporter/scripts/push-ai-daily-to-feishu.mjs examples/ai-daily-example.md
```

## Use The Skill

Copy `skills/feishu-automation-reporter` into your Codex skills directory, or keep it in a repository that your Codex setup can install from.

Use it when you want Codex to:

- add Feishu push support to a Markdown-producing automation;
- configure `.env.local` safely;
- create or adapt a Feishu card script;
- debug a missing push;
- validate card payloads with `FEISHU_DRY_RUN=1`.

## Automation Prompt Pattern

Add a final step to your Codex automation prompt:

```text
After generating the Markdown file, if FEISHU_WEBHOOK_URL is configured in the environment or .env.local, run:
node skills/feishu-automation-reporter/scripts/push-ai-daily-to-feishu.mjs <generated-markdown-file>
If it is not configured, only generate the file and state that Feishu was not pushed.
```

## Security Notes

- Never commit `.env.local`.
- Never paste a real webhook into prompts, issues, docs, examples, or screenshots.
- Rotate the webhook if it was exposed.
- Prefer `FEISHU_ENV_FILE` for worktree automations that cannot see your local secret file.
